# HD Cat Boot Animation Design

**Date:** 2026-04-06
**Status:** Approved

## Overview

Replace the text-character cat boot animation (14x7 sprite, drawn as '1' chars at FONT_5X8) with a pixel-art silhouette (56x56px, drawn at 1:1 pixel scale on the 128x64 SSD1306 OLED). Composable from shared body + variable tail (2 states) + variable legs (4 walk poses) = 8 frames total. Keep the deja vu glitch, scrolling ground line, and Matrix-style HEARTWOOD decrypt reveal.

## Approach

Standalone converter tool (Approach A). A Rust binary in `sprite-gen/` reads ASCII art source files, assembles all 8 frame combinations, and writes `firmware/src/cat_sprites.rs` with flat `const [u64; 56]` arrays. No build.rs changes -- ESP-IDF build system stays untouched.

## Source art format

ASCII art files in `docs/sprites/`, one per composable piece. `#` for on-pixel, `.` for off-pixel, padded to 56 columns.

The reference sprite (`docs/cat-sprite-reference.txt`) decomposes into three vertical zones:

| Zone | Rows | File(s) | Description |
|------|------|---------|-------------|
| Tail | 1-14 | `tail-up.txt`, `tail-down.txt` | Free tail above the body (2 variants) |
| Body | 15-44 | `body.txt` | Head, neck, torso, upper haunches (constant) |
| Legs | 45-56 | `legs-stride-{1,2,3,4}.txt` | Four separated legs (4 walk poses) |

Total: 14 + 30 + 12 = 56 rows per assembled frame.

Leg variants represent a four-phase walk cycle:
- stride-1: front-left + back-right forward
- stride-2: legs passing centre
- stride-3: front-right + back-left forward
- stride-4: legs passing centre (opposite phase)

Tail variants: curled upward (as in reference) and relaxed/hanging lower.

## Converter tool (sprite-gen)

Standalone crate at `sprite-gen/` (no workspace, same pattern as other crates). Run with `cd sprite-gen && cargo run`.

Steps:
1. Read all 7 source art files from `docs/sprites/`
2. Parse each into `Vec<u64>` (one u64 per row, bit 55 = leftmost column, bit 0 = rightmost)
3. Validate: tail files are 14 rows, body is 30 rows, legs are 12 rows, all 56 columns wide
4. Assemble 8 frames: 2 tails x 4 leg poses, concatenating tail + body + legs
5. Write `firmware/src/cat_sprites.rs`

Minimal dependencies -- just `std::fs` and string parsing.

## Output format (cat_sprites.rs)

```rust
/// Auto-generated by sprite-gen. Do not edit by hand.
/// Source: docs/sprites/{body,tail-*,legs-*}.txt

pub const FRAME_ROWS: usize = 56;
pub const FRAME_COLS: usize = 56;
pub const FRAME_COUNT: usize = 8;

/// Frames 0-3: tail up + 4 leg poses.
/// Frames 4-7: tail down + 4 leg poses.
pub const FRAMES: [[u64; FRAME_ROWS]; FRAME_COUNT] = [
    [ /* frame 0: tail-up + stride-1 */ ],
    // ...
];
```

## Firmware drawing code

New `draw_sprite_hd` function replaces the text-character `draw_sprite`:

```rust
fn draw_sprite_hd(
    display: &mut Display<'_>,
    frame: &[u64; 56],
    x_offset: i32,
    y_offset: i32,
) {
    for row in 0..56i32 {
        let bits = frame[row as usize];
        let py = y_offset + row;
        if py < 0 || py >= 64 { continue; }
        for col in 0..56i32 {
            if (bits >> (55 - col)) & 1 == 1 {
                let px = x_offset + col;
                if px >= 0 && px < 128 {
                    Pixel(Point::new(px, py), BinaryColor::On)
                        .draw(display).ok();
                }
            }
        }
    }
}
```

Cat is 56px wide on 128px screen. `x_offset` runs from -56 (fully off-left) to 128 (fully off-right). `y_offset` is 0 (cat flush to top, 8px spare at bottom for ground line).

The old `draw_sprite`, `CAT` arrays, and text-character machinery are removed. `cat_sprites.rs` added as `mod cat_sprites;`.

## Animation behaviour

### Walk cycle

| Step % 8 | Tail | Legs | Motion |
|-----------|------|------|--------|
| 0 | up | stride-1 | front-left forward |
| 1 | up | stride-2 | legs passing |
| 2 | up | stride-3 | front-right forward |
| 3 | up | stride-4 | legs passing (opposite) |
| 4 | down | stride-1 | front-left forward |
| 5 | down | stride-2 | legs passing |
| 6 | down | stride-3 | front-right forward |
| 7 | down | stride-4 | legs passing (opposite) |

Tail alternates every 4 steps (one full leg cycle = one tail sway).

### Timing

- x step: 2px per frame
- Frame delay: 45ms
- Total walk: x from -56 to 128 = 92 steps = ~4.1 seconds
- Lead-in/lead-out: 2 empty frames at 50ms each
- Phase 2 decrypt: 25 frames at 60ms = 1.5s (unchanged)

### Deja vu glitch

Triggers when cat reaches centre-screen (`x_offset` around 36, so cat midpoint is at pixel 64). Ghost cat appears 40px behind for 3 frames, using frame offset +4 (opposite tail state for uncanny effect).

### Ground line

Unchanged: scrolling dashes at y=63.

### Phase 2: HEARTWOOD decrypt reveal

Unchanged: screen clears, HEARTWOOD decrypts letter by letter with LFSR randomisation, version shown once resolved.
